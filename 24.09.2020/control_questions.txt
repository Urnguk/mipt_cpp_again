В случае встраиваемой функции, компилятор при ее выполнении вместо обращения к функции по ее адресу просто заменяет место в коде где она была вызвана ее телом. При этом стоит понимать, что объявление встраиваемой функции вообще говоря не значит, что компилятор так сделает. Это скорее подсказка компилятору, он сам решит будет ли данное действие полезно для повышения производительности.
Число аргументов по умолчанию меньше либо равно числу аргументов передаваемых в функцию. При это заполняться они должны "с конца", то есть ситуация когда за аргументом, имеющим значение по умолчанию идет аргумент, не имеющий значения по умолчанию, невозможна.
На основании типа переданных ей аргументов. Если тип переданных аргументов не соответствует ни одной из перегрухок функции, будет выбрана та перегрузка, для которой проще всего выполнить преобразование типов аргументов.
Для захвата переменных из внешней области памяти в лямбда функции существует специальный список захвата, а для обычных функций можно использовать static переменные.
Они не имеют пространства имен, а значит раз объявленный макрос уже не даст воспользоваться ничем с таким же именем, что может привести к непредсказуемой работе программы. Помимо этого сама логика макросов достаточно сложная может работать не совсем так как ожидалось(особенно наглядно это видно с арифметическими операциями, где для корректной работы макроса необходимо оборачивать все аргументы в скобки). Также их тяжело отлаживать. Поэтому стоит использовать только очень короткие и понятные макросы, а все остальное переносить в функции.
